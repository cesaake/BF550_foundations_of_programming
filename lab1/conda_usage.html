📘 Conda Environment Tutorial for Students

1️⃣ What is a Conda Environment?

A Conda environment is an isolated workspace that includes its own Python version and packages. This allows students and developers to avoid version conflicts and manage project-specific dependencies easily.

2️⃣ Creating a Conda Environment

Create an environment named TEZ with Python 3.10 and some useful packages like numpy, pandas, and jupyterlab:

conda create -n TEZ python=3.10 numpy pandas matplotlib scikit-learn jupyterlab -y

📌 -n TEZ gives the environment its name.
📌 -y auto-approves installation.
📌 jupyterlab is a common tool used by students for interactive coding.

3️⃣ Activating & Deactivating the Environment

# Activate the TEZ environment
conda activate TEZ

# Deactivate the current environment
conda deactivate

4️⃣ Listing and Removing Environments

# List all available Conda environments
conda env list

# Remove the TEZ environment
conda env remove -n TEZ

5️⃣ Creating Environment with YAML File

Instead of typing all packages in one line, you can use a environment.yml file:

🔹 Sample environment.yml

name: TEZ
channels:
  - conda-forge
  - defaults
dependencies:
  - python=3.10
  - numpy
  - pandas
  - matplotlib
  - scikit-learn
  - jupyterlab

🔹 Create Environment from YAML

conda env create -f environment.yml

🔹 Update Environment from YAML

conda env update -f environment.yml --prune

🔹 Export Current Environment

conda env export --name TEZ > environment.yml

6️⃣ Bonus: Clean Up Conda

Remove unused files and cache to save disk space:

conda clean --all
