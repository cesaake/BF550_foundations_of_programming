Miniconda vs Full Anaconda (Distribution Installers)

Miniconda is a minimal installer for Conda, while Anaconda Distribution includes Conda and hundreds of preinstalled packages.

Feature Miniconda Full Anaconda
Size ~50 MB ~3–4 GB
Packages Only Conda + Python Includes 250+ packages
Flexibility Highly customizable Preconfigured for data science
Best for Lightweight, custom setups Quick start for data science

Conda Environment Setup Tutorial

Here’s how to create and manage a Conda environment called TEZ with useful Python packages:

# Create environment with Python 3.10 and common packages
conda create -n TEZ python=3.10 numpy pandas matplotlib scikit-learn jupyterlab -y

# Activate environment
conda activate TEZ

# Deactivate environment
conda deactivate

# Remove environment
conda env remove -n TEZ

Using YAML Files for Reproducibility

Create an environment.yml file to share or reproduce environments:

name: TEZ
channels:
  - conda-forge
  - defaults
dependencies:
  - python=3.10
  - numpy
  - pandas
  - matplotlib
  - scikit-learn
  - jupyterlab

Commands to work with YAML files:

# Create env from YAML
conda env create -f environment.yml

# Update existing env
conda env update -f environment.yml --prune

# Export current env
conda env export --name TEZ > environment.yml

Basic Conda Commands

# List all environments
conda env list

# List installed packages in current environment
conda list

# Export list of packages
conda list --name TEZ --export > requirements.txt

# Update Conda itself
conda update conda

# Clean up caches
conda clean --all
